\subsection{Real-Time Backend Services}
A FastAPI-driven service architecture operates as the central controller, designed for high throughput and low latency.
\begin{itemize}
    \item \textbf{Data Ingestion \& Streaming:} Efficient handling of continuous video feeds connected through internal deployment channels, parsing streams frame-by-frame.
    \item \textbf{Asynchronous Dispatch:} Using WebSockets to instantaneously broadcast bounding box coordinates, unique tracking IDs, and system status markers back to the connected frontend clients without blocking the main event loops.
    \item \textbf{State Persistence:} Integrating PostgreSQL and TimescaleDB to securely log tracking metadata and system events, ensuring the analytics pipeline retains temporally accurate data.
\end{itemize}

\subsection{AI Models \& Inference Pipeline}
The intelligence driving the physical tracking relies on a multi-stage computer vision workflow integrated directly into the backend.
\begin{itemize}
    \item \textbf{Object Detection \& Spatial Tracking:} Utilizing YOLO-based models for high-speed pedestrian detection, tightly coupled with SORT-driven algorithms to maintain short-term, single-camera spatial continuity.
    \item \textbf{Cross-Camera Re-Identification (Re-ID):} An implementation of a Torchreid-based neural network architecture capable of extracting deep appearance embeddings using an OSNet-AIN backbone. This enables the system to re-identify subjects across disjointed camera views by comparing feature vectors, mitigating the limitations of blind spots.
    \item \textbf{Experiment Tracking \& Model Management:} Throughout the development lifecycle, MLflow was utilized to systematically log all model training experiments, track hyperparameters, and compare evaluation metrics across different iterations of the AI models. This ensured rigorous engineering practices and reproducibility before finalizing the weights for deployment.
\end{itemize}

\subsection{Continuous Integration and Deployment (CI/CD)}
To ensure code quality, prevent regressions, and streamline the development lifecycle, SpotOn employs automated Continuous Integration (CI) pipelines using GitHub Actions for both the frontend and backend repositories. These pipelines are triggered automatically upon every pull request and push to the main branches.

\begin{itemize}
    \item \textbf{Backend CI Pipeline:} The backend workflow is optimized for speed and isolation. It utilizes \texttt{uv}, an extremely fast Python package and project manager, to provision Python 3.12 environments and install dependencies. To keep the cloud runners efficient, the pipeline specifically requests CPU-only wheels for heavy machine learning libraries (like PyTorch) and avoids installing GPU-specific CUDA toolkits. Finally, it executes an extensive suite of over 160 isolated unit and integration tests using \texttt{pytest}, ensuring API contracts, Re-ID logic, and database operations remain fully functional.
    \item \textbf{Frontend CI Pipeline:} The frontend workflow utilizes Node.js and \texttt{npm} to ensure UI stability. The pipeline executes a strict progression of quality checks: running static code linting via ESLint to maintain styling conventions, enforcing rigorous TypeScript type-checking to catch compilation errors, and ultimately executing the suite of unit and component tests to validate UI rendering and state management logic.
\end{itemize}

\subsection{Re-Identification Performance}
\label{subsec:reid-results}

Table~\ref{tab:reid} details cross-camera Re-ID performance. The optimized SpotOn pipeline improves matching accuracy by confining the OSNet-AIN backbone evaluation strictly to predefined 5\% camera edge zones. By replacing exhaustive frame-by-frame feature extraction with geometry-first matching and edge-trigger Re-ID, the system actively filters out false positives caused by appearance ambiguity---such as uniform worker attire in factories. Consequently, the factory average Rank-1 accuracy improved by +3.4\%, reaching 72.8\%, while the campus environment saw a steeper +7.1\% gain due to higher variance in pedestrian clothing facilitating cleaner feature embeddings when extracted selectively.

\subsection{Overall Satisfaction}
Participant responses were overwhelmingly positive. Users consistently highlighted the responsiveness of the live video feed and praised the clarity of the 2D floor-plan overlay, which allowed non-technical operators to intuitively interpret real-time subject positions without prior training. No critical usability issues were reported, and all participants successfully navigated between the environment selection, group tracking view, and analytics pages without external assistance. The consensus among participants was that SpotOn provides a practical, reliable, and immediately deployable solution for facility-scale multi-camera surveillance.

\subsection{Specific Compliments}
Participants noted several standout qualities of the delivered system:

\begin{itemize}
    \item \textbf{Intuitive Interface:} Users appreciated the clean layout of the dashboard. The logical separation between the live tracking panel and the topographical map was described as immediately understandable, requiring no prior learning curve.
    \item \textbf{Real-Time Responsiveness:} The sub-50\,ms end-to-end processing latency was perceptible to end-users, who noted that bounding box updates on both the camera feed and the floor plan felt instantaneous and smooth during the session.
    \item \textbf{Cross-Camera Continuity:} Participants were notably impressed when a tracked individual seamlessly transitioned between camera views while retaining the same global identity, demonstrating the Re-ID pipeline's effectiveness in a live setting.
    \item \textbf{Analytics Clarity:} The historical analytics page received positive remarks for its zone traffic heatmaps and dwell-time charts, which users found directly actionable for operational planning and post-incident review.
    \item \textbf{System Stability:} Throughout the entire demonstration, no crashes, frame freezes, or connectivity errors were observed, reinforcing operator confidence in the reliability of the deployed WebSocket and streaming infrastructure.
\end{itemize}

\subsection{User Feedback}
In addition to verbal observations, participants provided written feedback at the conclusion of the session. A representative response is reproduced below:

\begin{quote}
\textit{``The system is impressively easy to use. I was able to follow individuals across different cameras without any confusion, and the map view made the whole tracking concept very clear. I would feel confident using this in a real facility. My only suggestion would be to add configurable alert thresholds so that the system can automatically notify supervisors when a specific zone exceeds a certain occupancy level.''}
\end{quote}

This suggestion directly aligns with a planned enhancement in the future development roadmap: the introduction of configurable zone-occupancy alert triggers and automated notification dispatch to registered operator accounts. Overall, the acceptance evaluation confirms that the SpotOn system meets its core usability objectives and is well-positioned for pilot deployment in real-world security and facility management environments.
